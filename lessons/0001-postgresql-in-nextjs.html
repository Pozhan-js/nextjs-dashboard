PostgreSQL 在这个项目里做什么？

目标：看懂 PostgreSQL、Neon、Vercel 和项目代码之间的关系，并读懂一条真实查询。

一句话模型

PostgreSQL 是数据库软件；Neon 是替你运行 PostgreSQL 的云服务；Vercel 运行 Next.js；项目里的 postgres 包负责让代码与数据库通信。

浏览器 → Next.js/Vercel → postgres 客户端 → POSTGRES_URL 指向的 Neon → PostgreSQL 表

四个角色不要混淆

名称作用类比
PostgreSQL保存、查询并保护结构化数据数据库引擎
Neon托管 PostgreSQL，提供地址、账号、备份等能力数据库房东
Vercel部署并运行 Next.js 应用应用服务器
postgresNode.js 连接 PostgreSQL 的驱动翻译与通信工具

数据是怎样组织的

PostgreSQL 属于关系型数据库。数据主要放在表中；表有列，实际记录是行。这个项目有 customers、invoices、users 和 revenue 等表。

  • 主键：一行数据的唯一身份证，例如 invoices.id。
  • 关联字段：一张表用某列关联另一张表，例如 invoices.customer_id 与客户 ID 对应。当前项目尚未为它声明真正的 FOREIGN KEY 约束。
  • 约束：数据库自己拒绝非法数据，例如 NOT NULL、UNIQUE。

读懂项目里的一条 SQL

SELECT invoices.amount, customers.name
FROM invoices
JOIN customers ON invoices.customer_id = customers.id
WHERE invoices.amount = 666;
  • SELECT：要哪些列。
  • FROM：从哪张表开始。
  • JOIN：按客户 ID 把发票和客户拼起来。
  • WHERE：只保留金额等于 666 的记录。

连接字符串是什么

postgresql://用户名:密码@主机地址/数据库名

项目通过环境变量 POSTGRES_URL 读取它。它相当于数据库钥匙，不能写进代码或提交到 Git。Vercel 环境变量只负责安全地把这把钥匙交给运行中的应用。

以后会经常遇到的三个能力

  1. 事务：多步修改要么全部成功，要么全部撤销。
  2. 索引：像书的目录，加快查询，但会占空间并增加写入成本。
  3. 约束：把关键数据规则交给数据库兜底。

当前阶段先会读表和 SQL。等真实查询变慢时再学索引，等一次操作包含多个写入时再深入事务。

一分钟自测

如果删除 Vercel 项目但保留 Neon 数据库，数据一定会消失吗？

查看答案

不一定。Vercel 负责运行应用，Neon 托管数据库；二者是不同资源。只有删除 Neon 数据库或其中的数据，数据才会消失。
